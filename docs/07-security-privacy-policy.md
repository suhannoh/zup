# 07. 보안·개인정보·법적 리스크 정책

## 1. 기본 원칙

이 서비스는 사용자의 개인정보를 거의 수집하지 않는 구조를 우선한다.

MVP 원칙:

- 사용자는 로그인하지 않는다.
- 생일 입력을 요구하지 않는다.
- 혜택 조회는 익명으로 가능하다.
- 제보 이메일은 선택 입력이다.
- 브랜드 공식 서비스로 오해되지 않게 한다.
- 공식 출처와 최근 확인일을 표시한다.
- 로고/이미지 사용은 최소화한다.

## 2. 수집하는 정보

### 자동 수집 가능 정보

- 페이지 조회수
- 조회 대상
- 조회 날짜
- 기본 로그

개인 식별 목적이 아니다.

### 사용자가 직접 입력할 수 있는 정보

제보 폼:

- 브랜드명
- 제보 유형
- 제보 내용
- 참고 링크
- 이메일 선택 입력

이메일은 답변이 필요할 경우에만 선택적으로 받는다.

## 3. 수집하지 않는 정보

MVP에서 수집하지 않는다.

- 사용자 생일
- 이름
- 전화번호
- 주소
- 결제 정보
- 로그인 정보
- 민감 정보
- 위치 정보

## 4. 제보 이메일 처리

이메일은 선택 입력이다.

정책:

- 이메일 없이 제보 가능
- 이메일은 제보 처리 회신 목적
- 외부에 공개하지 않음
- 관리자만 확인 가능
- 추후 삭제 요청 가능하게 고려

## 5. 브랜드 정보 사용 정책

브랜드명은 정보 제공 목적으로 사용한다.

주의:

- 서비스가 공식 브랜드 서비스처럼 보이면 안 된다.
- “공식”이라는 표현은 출처에만 사용한다.
- “공식 확인 링크”는 가능하지만 “공식 제휴”처럼 오해될 표현은 금지한다.
- 브랜드 로고는 MVP에서 사용하지 않는다.
- 필요 시 텍스트/이니셜 뱃지로 대체한다.

면책 문구 예시:

```text
본 서비스는 각 브랜드의 공식 서비스가 아니며, 공개된 공식 출처를 바탕으로 혜택 정보를 정리한 독립 정보 제공 서비스입니다.
브랜드 정책은 수시로 변경될 수 있으므로 사용 전 공식 페이지 또는 앱에서 최종 확인해 주세요.
```

## 6. 로고/이미지 리스크

MVP에서는 실제 브랜드 로고를 사용하지 않는다.

허용 방향:

- 브랜드명 텍스트
- 이니셜 뱃지
- 단색 배지
- 직접 제작한 일반 아이콘

피해야 할 것:

- 브랜드 로고 이미지 무단 저장
- 공식 앱 화면 캡처 공개
- 브랜드 이미지 무단 사용
- 광고 소재 무단 사용

## 7. 크롤링 리스크

처음부터 자동 크롤링을 전면 도입하지 않는다.

금지:

- 로그인 필요한 페이지 크롤링
- 앱 내부 데이터 무단 수집
- 약관에서 금지한 자동 수집
- 과도한 요청
- 이미지 무단 저장

가능한 방향:

- 공개 공식 페이지 URL 저장
- 관리자가 수동 확인
- robots.txt 확인 후 낮은 빈도 변경 감지
- 본문 hash만 저장
- 변경 감지 후 자동 반영하지 않고 관리자 검수

## 8. 정보 정확성 리스크

주요 위험:

- 혜택이 종료됐는데 노출됨
- 조건이 바뀜
- 지점별로 다름
- 등급별 조건이 누락됨
- 생일 등록 시점 조건이 누락됨
- 앱 전용 혜택인데 앱 필요 표시가 없음

대응:

- 최근 확인일 표시
- 공식 출처 링크 표시
- 사용자 제보 버튼
- 검수 상태 관리
- 90일 이상 미확인 자동 표시
- 조회수 높은 브랜드 우선 재검수
- 면책 문구

## 9. 관리자 보안

관리자 기능은 추후 인증을 적용한다.

권장:

- 관리자 로그인
- 비밀번호 해시 저장
- 관리자 API 인증
- 관리자 계정 최소화
- 운영 로그 기록

MVP 초기 skeleton에서는 permitAll로 시작할 수 있지만, 공개 배포 전에는 관리자 인증을 적용해야 한다.

## 10. 공개 전 체크리스트

- 브랜드 로고 무단 사용 여부 확인
- 공식 출처 링크 표시 여부 확인
- 최근 확인일 표시 여부 확인
- 면책 문구 표시 여부 확인
- 제보 이메일 선택 입력 여부 확인
- 관리자 API 인증 적용 여부 확인
- robots.txt / sitemap.xml 설정 확인
