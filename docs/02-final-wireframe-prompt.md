# 02. 최종 와이어프레임 생성 프롬프트

이 문서는 UI/UX 와이어프레임을 생성하거나 다른 AI/디자인 도구에 전달하기 위한 프롬프트다.

## 프롬프트

너는 모바일 우선 정보 큐레이션 웹서비스의 UX/UI 디자이너다.

설명문이 아니라, **Zup MVP의 실제 화면 흐름 와이어프레임**을 만들어라.

## 서비스 설명

서비스명: Zup

서비스 정의:
브랜드별 생일 쿠폰, 무료 혜택, 할인, 멤버십 혜택을 공식 출처 기준으로 정리하고, 사용자가 조건별로 빠르게 필터링해서 볼 수 있는 검색 유입형 정보 큐레이션 웹서비스.

## 핵심 사용자

생일 전후로 아래 키워드를 검색하는 사용자.

- 생일 혜택
- 생일 쿠폰
- 스타벅스 생일 쿠폰
- 올리브영 생일 혜택
- 영화관 생일 혜택
- 카페 생일 혜택

## UI 방향

- 모바일 우선
- 로그인 없이 바로 정보 확인
- 블로그보다 빠르게 비교 가능한 카드형 UI
- 정보 신뢰도를 보여주는 최근 확인일/공식 출처 표시
- 브랜드 로고 대량 사용 금지
- 브랜드 컬러/이니셜 뱃지 중심
- 광고성 느낌보다 정보 서비스 느낌
- 과한 앱 UI보다 SEO형 웹페이지 느낌

## 필요한 화면

### 1. 메인 페이지

목표:
5초 안에 서비스가 뭔지 이해되어야 한다.

구성:

- 상단 헤더
  - 서비스명
  - 브랜드
  - 카테고리
  - 제보하기
- 히어로 카피
  - “생일에 받을 수 있는 브랜드 혜택, 조건별로 한 번에 보기”
- 검색창
  - placeholder: “브랜드명을 검색해보세요. 예: 스타벅스, 올리브영, CGV”
- 빠른 필터 버튼
  - 무료 혜택
  - 앱 불필요
  - 생일월 가능
  - 카페
  - 영화관
  - 뷰티
- 인기 생일 혜택 카드
- 최근 업데이트 카드
- 안내 문구
  - “모든 정보는 공식 출처 기준으로 확인하며, 브랜드 정책에 따라 변경될 수 있습니다.”

### 2. 브랜드 목록 페이지

목표:
사용자가 조건별로 혜택을 빠르게 걸러본다.

구성:

- 검색창
- 카테고리 필터
- 조건 필터
  - 무료
  - 할인
  - 쿠폰
  - 앱 필요
  - 앱 불필요
  - 멤버십 필요
  - 조건 없음
  - 생일 당일
  - 생일월
- 정렬
  - 인기순
  - 최근 확인순
  - 이름순
- 브랜드 카드 리스트

브랜드 카드에 표시할 정보:

- 브랜드명
- 카테고리
- 혜택 요약
- 태그 배지
- 최근 확인일
- 공식 확인 배지

### 3. 브랜드 상세 페이지

목표:
SEO 핵심 페이지이자 사용자가 실제로 혜택 조건을 확인하는 페이지.

예시 제목:
“스타벅스 생일 쿠폰 조건 및 사용 기간”

구성:

- 브랜드명
- 혜택 요약 박스
- 핵심 조건 배지
  - 무료
  - 앱 필요
  - 생일월
  - 멤버십 필요
- 혜택 받는 방법
- 사용 가능 기간
- 주의사항
- 공식 출처 링크
- 최근 확인일
- 비슷한 브랜드 추천
- 정보 수정 제보 버튼
- 면책 문구

### 4. 카테고리 페이지

예시:

- 카페 생일 혜택 모음
- 외식 브랜드 생일 쿠폰 모음
- 영화관 생일 혜택 모음
- 뷰티 브랜드 생일 쿠폰 모음

구성:

- 카테고리 설명
- 대표 혜택 TOP
- 필터
- 브랜드 리스트
- 관련 태그 링크

### 5. 태그 페이지

예시:

- 앱 설치 없이 받을 수 있는 생일 혜택
- 무료 생일 쿠폰 모음
- 생일월 전체 사용 가능한 혜택
- 조건 없는 생일 혜택

구성:

- 태그 설명
- 해당 조건에 맞는 혜택 리스트
- 관련 카테고리
- 관련 태그

### 6. 정보 수정 제보 페이지

구성:

- 제보 유형 선택
  - 정보가 틀렸어요
  - 혜택이 종료됐어요
  - 조건이 달라졌어요
  - 새 혜택이 있어요
  - 공식 링크를 찾았어요
- 브랜드명
- 제보 내용
- 참고 링크
- 이메일 선택 입력
- 제출 버튼

### 7. 관리자 페이지

디자인은 간단해도 된다. 기능 중심.

구성:

- 대시보드
  - 게시 중 혜택 수
  - 검수 필요 혜택 수
  - 90일 이상 미확인 혜택 수
  - 최근 제보 수
- 브랜드 관리
- 혜택 관리
- 제보 관리
- 검수 필요 목록
- 오래된 데이터 목록

## 디자인 톤

- 색상: 밝고 깔끔한 정보 서비스
- 배경: #FAFAFA 또는 유사한 밝은 톤
- 카드: 흰색, 둥근 모서리, 얕은 그림자
- 배지: 조건을 빠르게 구분할 수 있는 작은 pill 형태
- 폰트: 가독성 우선
- CTA: 과하게 튀지 않게

## 반드시 피할 것

- 쇼핑몰처럼 과한 광고 느낌
- 브랜드 로고를 대량으로 사용하는 UI
- 사용자가 회원가입해야 할 것 같은 UI
- 앱처럼 복잡한 온보딩
- AI 추천 강조
- 쿠폰을 직접 발급하는 것처럼 오해되는 표현

## 최종 산출물

- 모바일 화면 6~8개
- 데스크톱 화면 2~3개
- 각 화면의 핵심 컴포넌트 설명
- 사용자 흐름
- MVP에서 꼭 필요한 화면과 나중에 만들 화면 구분
