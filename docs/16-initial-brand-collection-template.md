# 16. 초기 브랜드 수집 스프레드시트 템플릿

## 1. 목적

이 문서는 Zup MVP 운영을 위한 초기 브랜드 30개 수집 스프레드시트 컬럼을 정의한다.

실제 혜택 내용은 공식 출처 확인 후 입력해야 한다. 블로그, 커뮤니티, 개인 후기에서 발견한 내용은 후보 또는 힌트로만 기록한다.

## 2. 스프레드시트 컬럼

| 컬럼명 | 설명 | 예시 |
| --- | --- | --- |
| 카테고리 | 브랜드 카테고리 | 카페 |
| 브랜드명 | 사용자에게 표시할 브랜드명 | 스타벅스 |
| 브랜드 slug | URL에 사용할 영문 slug | starbucks |
| 공식 홈페이지 URL | 브랜드 공식 홈페이지 | https://www.example.com |
| 멤버십 URL | 멤버십 안내 URL | https://www.example.com/membership |
| 앱 URL | 공식 앱 안내 URL | https://www.example.com/app |
| 생일 혜택 존재 여부 | 있음/없음/확인 필요 | 확인 필요 |
| 혜택명 | 혜택 제목 | 생일 쿠폰 |
| 혜택 요약 | 짧은 설명 | 생일월 쿠폰 제공 |
| 혜택 유형 | FREE_ITEM/DISCOUNT/COUPON/POINT/GIFT/UPGRADE/ETC | COUPON |
| 생일 사용 기간 유형 | BIRTHDAY_ONLY/BIRTHDAY_MONTH/BEFORE_AFTER_DAYS/ISSUED_BEFORE_BIRTHDAY/UNKNOWN | UNKNOWN |
| 앱 필요 여부 | true/false/확인 필요 | true |
| 회원가입 필요 여부 | true/false/확인 필요 | true |
| 멤버십 필요 여부 | true/false/확인 필요 | true |
| 구매 조건 여부 | true/false/확인 필요 | 확인 필요 |
| 조건 요약 | 조건을 짧게 정리 | 멤버십 가입 필요 |
| 공식 출처 URL | 게시 근거가 되는 공식 URL | https://www.example.com/faq |
| 출처 유형 | SourceType 값 | OFFICIAL_FAQ |
| 최근 확인일 | 운영자가 확인한 날짜 | 2026-04-28 |
| 검수 상태 | DRAFT/NEEDS_CHECK/VERIFIED/PUBLISHED/EXPIRED/HIDDEN | DRAFT |
| 비고 | 모호한 점, 추가 확인 사항 | 앱 화면 재확인 필요 |

## 3. 초기 수집 우선 카테고리

- 카페
- 베이커리
- 외식
- 영화·문화
- 뷰티
- 테마파크

## 4. 초기 후보 브랜드 예시

| 카테고리 | 후보 브랜드 |
| --- | --- |
| 카페 | 스타벅스, 투썸플레이스, 이디야 |
| 베이커리 | 파리바게뜨, 뚜레쥬르 |
| 외식 | 아웃백, 빕스 |
| 영화·문화 | CGV, 메가박스, 롯데시네마 |
| 뷰티 | 올리브영, 이니스프리 |
| 테마파크 | 롯데월드, 에버랜드 |
| 통합 멤버십 | CJ ONE, 해피포인트 |

## 5. 수집 작업 순서

1. 후보 브랜드를 카테고리별로 정리한다.
2. 브랜드 공식 홈페이지와 멤버십 URL을 찾는다.
3. 생일 혜택 존재 여부를 공식 출처에서 확인한다.
4. 혜택 조건을 임의로 추정하지 않고 출처에 있는 내용만 기록한다.
5. 공식 출처가 없으면 `검수 상태`를 `DRAFT` 또는 `NEEDS_CHECK`로 둔다.
6. 공식 출처 URL과 최근 확인일을 기록한다.
7. 관리자 화면에 입력하기 전 중복 브랜드와 중복 혜택을 확인한다.

## 6. 입력 주의사항

- 실제 혜택 내용은 공식 출처 확인 후 입력한다.
- 블로그/커뮤니티 정보는 공식 출처 확인 전까지 게시 근거로 사용하지 않는다.
- 조건이 모호한 경우 `확인 필요`로 남기고 `PUBLISHED` 처리하지 않는다.
- 혜택이 종료되었거나 현재 제공 여부가 불확실하면 `EXPIRED` 또는 `NEEDS_CHECK`로 관리한다.
