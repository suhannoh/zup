# 01. MVP 범위 정의

## 1. MVP 목표

Zup의 1차 MVP는 “완성된 혜택 포털”이 아니라 **검색 유입을 실험할 수 있는 최소 정보 서비스**다.

핵심 목표는 다음이다.

1. 사용자가 로그인 없이 브랜드별 생일 혜택을 볼 수 있다.
2. 사용자가 조건별로 혜택을 필터링할 수 있다.
3. 브랜드 상세 페이지가 SEO-friendly URL로 제공된다.
4. 관리자가 혜택 정보를 등록/수정/검수할 수 있다.
5. 각 혜택에 공식 출처와 최근 확인일을 표시한다.
6. 사용자가 정보 오류를 제보할 수 있다.

## 2. MVP 포함 기능

### 공개 사용자 기능

- 메인 페이지
- 브랜드 목록
- 브랜드 상세 페이지
- 카테고리별 혜택 페이지
- 태그/조건별 혜택 페이지
- 브랜드명 검색
- 조건 필터
- 정보 수정 제보
- 공식 출처 링크 표시
- 최근 확인일 표시

### 관리자 기능

- 관리자 로그인
- 브랜드 등록/수정/비활성화
- 혜택 등록/수정/비활성화
- 혜택 검수 상태 변경
- 공식 출처 URL 등록
- 최근 확인일 수정
- 제보 목록 확인
- 제보 상태 변경
- 검수 필요 목록 확인
- 오래된 데이터 목록 확인

### 운영 기능

- 조회수 집계
- 인기 브랜드/혜택 기준 데이터 준비
- 최근 확인일 90일 초과 데이터 식별
- sitemap.xml
- robots.txt
- SEO title/description

- 실제 쿠폰 발급

- 결제 기능

## 4. 1차 데이터 범위

### 브랜드 수

- 최소 공개 기준: 20개
- 1차 목표: 30개
- 2차 목표: 50개

### 카테고리

MVP에서는 아래 6개만 사용한다.

- 카페
- 베이커리
- 외식
- 영화·문화
- 뷰티
- 테마파크

### 태그

초기 태그는 아래 기준으로 시작한다.

- 무료
- 할인
- 쿠폰
- 앱 필요
- 앱 불필요
- 회원가입 필요
- 멤버십 필요
- 조건 없음
- 최소 구매 필요
- 생일 당일
- 생일월
- 생일 전후 7일
- 공식 확인
- 확인 필요

## 6. MVP 성공 기준

### 개발 성공 기준

- 로컬에서 backend/frontend 실행 가능
- 공개 페이지 기본 조회 가능
- 관리자에서 브랜드/혜택 등록 가능
- 브랜드 상세 페이지가 slug 기반으로 접근 가능
- sitemap.xml 생성 가능
- 제보 등록 가능

### 운영 성공 기준

- 공식 출처가 있는 브랜드 30개 등록
- 각 혜택에 최근 확인일 표시
- Search Console 등록
- Naver Search Advisor 등록
- 최소 10개 이상 브랜드 상세 페이지 인덱싱 시도
- 실제 검색 유입 여부 측정 시작
