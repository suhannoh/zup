# 15. 공식 출처 검수 가이드

## 1. 목적

이 문서는 Zup에 입력되는 생일 혜택 정보의 신뢰도를 유지하기 위한 공식 출처 검수 기준을 정리한다.

Zup은 공식 출처로 검증 가능한 혜택만 사용자에게 공개하는 것을 원칙으로 한다.

## 2. 출처 우선순위

### 1순위: 게시 근거로 사용 가능

- 공식 홈페이지
- 공식 멤버십 안내
- 공식 FAQ
- 공식 공지사항

1순위 출처는 혜택명, 조건, 사용 기간, 멤버십 조건을 확인하는 기본 근거로 사용한다.

### 2순위: 보조 게시 근거로 사용 가능

- 공식 앱 내 쿠폰/멤버십 화면
- 공식 SNS
- 고객센터 답변

2순위 출처는 공식 채널임을 확인할 수 있을 때 게시 근거로 사용할 수 있다.

앱 화면이나 고객센터 답변은 URL로 검증하기 어려울 수 있으므로, 운영 메모에 확인 경로와 확인일을 남긴다.

### 3순위: 게시 근거로 사용 불가

- 블로그
- 커뮤니티
- 인스타 저장형 콘텐츠
- 개인 후기
- 비공식 혜택 모음 글

3순위 출처는 게시 근거로 사용하지 않는다.

3순위 출처는 브랜드 후보 또는 조건 변경 힌트로만 사용한다. 3순위 출처에서 발견한 내용은 반드시 1순위 또는 2순위 출처로 다시 확인한다.

## 3. 검수 정책

- 공식 출처가 없는 혜택은 `PUBLISHED`로 변경하지 않는다.
- 공식 출처가 모호하면 `NEEDS_CHECK` 또는 `DRAFT`로 둔다.
- 블로그/커뮤니티 정보는 수집 힌트로만 사용한다.
- 공식 출처의 조건과 사용자 제보가 충돌하면 혜택을 `NEEDS_CHECK`로 두고 재검수한다.
- 혜택 종료가 공식적으로 확인되면 `EXPIRED`로 변경한다.
- 오래된 혜택은 최근 확인일을 기준으로 재검수한다.

## 4. 상태 기준

| 상태 | 의미 | 운영 기준 |
| --- | --- | --- |
| `DRAFT` | 입력 중 | 브랜드/혜택 기본 정보 입력 단계 |
| `NEEDS_CHECK` | 공식 확인 필요 | 공식 출처가 부족하거나 사용자 제보로 재검수가 필요한 상태 |
| `VERIFIED` | 공식 확인 완료 | 공식 출처로 확인했으나 사용자 공개 전 상태 |
| `PUBLISHED` | 사용자 공개 | 공식 출처 확인 완료 후 사용자 화면 공개 상태 |
| `EXPIRED` | 종료 확인 | 혜택 종료가 공식 출처로 확인된 상태 |
| `HIDDEN` | 숨김 | 운영상 사용자 화면에 노출하지 않는 상태 |

## 5. 출처 유형별 입력 기준

| SourceType | 사용 기준 |
| --- | --- |
| `OFFICIAL_HOME` | 브랜드 공식 홈페이지 |
| `OFFICIAL_APP` | 공식 앱 내 쿠폰, 멤버십, 이벤트 화면 |
| `OFFICIAL_MEMBERSHIP` | 공식 멤버십 안내 페이지 |
| `OFFICIAL_FAQ` | 공식 FAQ |
| `OFFICIAL_NOTICE` | 공식 공지사항 또는 이벤트 안내 |
| `OFFICIAL_SNS` | 브랜드 공식 SNS 게시물 |
| `CUSTOMER_CENTER` | 고객센터 답변 또는 상담 결과 |
| `BLOG_REFERENCE` | 비공식 블로그 참고 자료 |
| `COMMUNITY_REFERENCE` | 비공식 커뮤니티 참고 자료 |

`BLOG_REFERENCE`, `COMMUNITY_REFERENCE`는 게시 근거가 아니다.

## 6. 검수 메모 작성 기준

검수 메모에는 운영자가 나중에 판단 근거를 재확인할 수 있도록 다음 내용을 남긴다.

- 확인한 공식 출처
- 확인한 날짜
- 주요 조건
- 모호한 부분
- 사용자 제보 반영 여부

메모 예시:

```text
공식 멤버십 안내 페이지에서 생일월 쿠폰 발급 조건 확인. 앱 필요, 멤버십 가입 필요. 2026-04-28 확인.
```

## 7. 게시 전 최종 판단

`PUBLISHED`로 변경하기 전에 다음 질문에 모두 "예"라고 답할 수 있어야 한다.

- 공식 출처로 혜택 존재를 확인했는가?
- 혜택 조건을 과장하거나 추정하지 않았는가?
- 사용 기간 또는 발급 기간을 명확히 썼는가?
- 앱/멤버십/구매 조건을 정확히 표시했는가?
- 최근 확인일을 입력했는가?
- 출처 URL 또는 확인 경로를 남겼는가?
